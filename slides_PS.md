# PowerShellの魅力を知ろう
## なぜ今 PowerShell なのか
- Windowsに標準搭載されている
- 管理作業を自動化できる
- 学習コストが比較的低い

PowerShellは「管理者のためのシェル」として進化してきました。
日常作業をコードで再現できるのが最大の強みです。
---
# 一貫したオブジェクト指向
## テキストではなくオブジェクト
- コマンドの出力がオブジェクト
- パイプラインで安全に連携
- 文字列処理の地獄から解放
Linux系シェルとの最大の違いはここです。
情報を「構造」として扱える安心感があります。
---
# パイプラインの考え方
## 処理をつなげる美しさ
- 小さなコマンドを組み合わせる
- 可読性が高い
- 再利用しやすい
Get → Filter → Action
この流れを自然に書けるのがPowerShellです。
---
# PowerShellの世界観
## 処理の流れを図で見る
   [ Cmdlet ]
        |
        v
   [ Object ]
        |
        v
   [ Cmdlet ]
        |
        v
    [ Result ]
コマンドはオブジェクトを受け取り、次へ渡します。

# 管理作業との相性
## IT管理者の強い味方
- Active Directory操作
- ファイル・サービス管理
- ログ収集と分析
GUIでの「手作業」をそのまま自動化できます。
一度書けば、何度でも同じ品質で実行可能です。
